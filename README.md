# Security+ SY0-701 — 3-Day Study Map

A streamlined, interactive web-based study tool designed to help candidates navigate the **CompTIA Security+ (SY0-701)** exam objectives in an accelerated 3-day format.

**Live Demo:** [https://sec-plus.netlify.app/](https://sec-plus.netlify.app/)

## 🚀 Overview
Preparing for the Security+ exam can be overwhelming due to the sheer volume of acronyms and domains. This project provides a structured roadmap that breaks down the official exam objectives into manageable daily milestones. 

Whether you are doing a final review or a deep dive into specific domains, this tool helps track progress across the five core pillars:
1. **General Security Concepts**
2. **Threads, Vulnerabilities, and Mitigations**
3. **Security Architecture**
4. **Security Operations**
5. **Security Program Management and Oversight**

## ✨ Features
- **Interactive Roadmap:** A visual breakdown of the SY0-701 domains.
- **Accelerated Timeline:** Structured specifically for a high-intensity 3-day study sprint.
- **Responsive Design:** Optimized for study sessions on desktop, tablet, or mobile.
- **CI/CD Integration:** Automatically updated via GitHub and Netlify.

## 🛠️ Tech Stack
- **Frontend:** HTML5, CSS3
- **Deployment:** [Netlify](https://www.netlify.com/)
- **CI/CD:** Automated deployment pipeline via GitHub-to-Netlify integration.

## ⚙️ DevOps & CI/CD Workflow
This project serves as a practical implementation of **DevSecOps** principles:
- **Continuous Integration:** GitHub acts as the version-controlled repository for all code changes.
- **Continuous Deployment:** Every `git push` to the `main` branch triggers an automatic deployment, ensuring the live study tool is always up to date.
- **Change Management:** All updates are tracked through commit history, providing an audit trail of modifications.

## 📖 How to Use
1. **Clone the repo:** `git clone https://github.com/JesseFlip/sec_plus.git`
2. **Local Preview:** Open `index.html` in any modern web browser.
3. **Updates:** Edit the `index.html` file, commit your changes, and push to GitHub to see the live site update via the CI/CD pipeline.
